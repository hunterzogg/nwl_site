# NWL Site — Handoff Document

## Project Overview

Static website for the National Wayzata League (NWL), a 13-season dynasty-style fantasy football league. Built with vanilla HTML/CSS/JS, hosted via GitHub Pages or Netlify (not yet deployed). All data is stored as JSON files loaded via `fetch()` — requires a local server or real hosting to run (double-clicking HTML files won't work due to browser `file://` security restrictions).

**To run locally:**
```
cd nwl_site
python3 -m http.server 8000
# then open http://localhost:8000
```

## Pages — Current State

### index.html ✅
Scorebug showing 3 stats (seasons, total points scored all-time, matchups), a distinct amber "2026 Season" banner linking to the current-season hub, nav cards to all 6 historical-archive sections, NWL shield logo. Subtitle copy is deliberately grand ("Thirteen years of rivalries, dynasties, and legendary collapses...") rather than utilitarian.

### season-2026.html ✅
The current-season hub — visually separate from the historical archive (see Phase 2 section below for the full design rationale). Four tabs: Matchups, Standings, Power Rankings, Commentary. Reads from `data/season_2026/*.json`, populated by `scripts/fetch_espn_week.py`. Empty states everywhere until the season starts.

### hall-of-fame.html ✅ (formerly lookup.html)
Three tabs: League Records (single-game, season, streaks), Head-to-Head (pick 2 managers → full matchup history), Seasons (weekly scores + standings + a top-3/last-place header, per year). Career Stats tab removed — moved to Managers page. Seasons tab shows two weekly-score tables — Regular Season and Playoffs (playoff weeks/managers vary correctly by era) — and standings grouped by division, sorted by wins.

### managers.html ✅
One card per manager (13 total), click to expand. Tenure + season count computed dynamically (no hardcoded dates). Former managers (Stowe, 2013-2016) get a badge and sort last. Expanded view shows: career stats (titles, sackos, W-L record, career/playoff PPG, playoff trips — trips only count top-6 finishes), full team name history (all 13 managers, extracted from All Time Team Ranks), Favorite Players (highest total roster-appearance count across draft + waiver/FA + trades — not draft capital), top 5 waiver/FA pickups by FAAB bid, 5 most recent trade acquisitions. Profile picture support via `assets/img/managers/{name}.jpg` (falls back to colored initials circle, no photos added yet). **Favorite NFL team is still empty for everyone** — not in NWL_Master.xlsx anywhere, hand-edit `data/manager_profiles.json` to add it.

### scorigami.html ✅
SVG scatter grid of every unique winning × losing score combo in league history (no filtering). Hover tooltip: count + most recent occurrence. Stat strip: unique scores, happened once, most common, highest combined.

### rankings.html ✅ (renamed "Team Rankings" in nav)
Per-team-season rankings (not career averages). Regular season and playoffs in separate tabs. Three sort toggles: Adjusted PPG, Raw PPG, and Opponent PPG (Regular Season tab only — no opponent-PPG source data for playoffs). Regular Season tab shows 5 PPG columns (adj/raw scored, adj/raw opponent). Playoffs tab has a "Reg. Rank − Playoff Rank" column (fixed rank, unaffected by the current filter/sort) — green when the playoff rank beat the regular-season rank, red when it didn't. Manager filter + season filter on each tab.

### draft.html ✅
Three tabs:
- **Draft Board**: filterable by season, manager, position, NFL team, player name search. NFL teams capitalized. Positions color-coded.
- **Round 1-3 Strategy**: filterable by manager and year (defaults to most recent). Shows finish column for each season. Summary table of strategy effectiveness (uses, avg finish, championships, sackos).
- **Draft Slot Analysis**: avg PPG, avg finish, championships, sackos by draft position. Sourced from Mgr Summary A84 (authoritative). Best avg PPG highlighted green, worst highlighted red.

### transactions.html ✅
Three tabs:
- **Transaction Log**: filterable by season, manager, type, player search. Losing waiver bids nest under winning bid when exact dates match. **2023-2025 are 100% exact-dated** (Rounds 9-10), year-only for 2013-2022 (no date source exists for those years in the workbook).
- **Trade Database**: filterable by season, manager, and now player name search. 2022 is year-only precision, 2023-2025 have exact dates. Player format: "POS - Player Name".
- **Summary**: historical league totals by season (from Mgr Summary A35, covers all 2013-2025), per-manager career counts (now includes Most/Fewest transactions in a season, with the year), acquisition type breakdown.

### Phase 2 — Live ESPN data (2026 season) ✅ pipeline + page built, not yet live
Design decided, fetch pipeline built and validated against real data, `pages/season-2026.html`
built. Nothing left to build - just waiting on the actual 2026 season to start (September) so
there's real data to publish.

**Decisions locked in:**
- Nav: new top-level "2026 Season" section (not a homepage takeover), same one-page-with-tabs
  pattern as every other section (Hall of Fame, Draft, Transactions).
- Four tabs: Matchups, Standings (both mechanical, no review needed), Power Rankings and
  Commentary (editorial - drafted by the fetch script, then reviewed/edited by Hunter before
  being marked `published: true`).
- Publishing mechanism: no backend, no admin page. A script pulls ESPN data into JSON files each
  week; Hunter reviews/edits the editorial ones directly; everything gets committed and the
  static site redeploys. The review step *is* the approval gate.
- ESPN auth can never live in client-side JS (private league - the espn_s2/SWID cookie would be
  stealable from anyone viewing page source), so even the mechanical pages go through the same
  script → JSON → redeploy path, not a live browser fetch.

**Built this session:**
- `scripts/fetch_espn_week.py` - pulls matchups + standings from ESPN (league 39276, confirmed
  private via a direct API test: `401 AUTH_LEAGUE_NOT_VISIBLE` from
  `lm-api-reads.fantasy.espn.com`) and writes `data/season_2026/matchups.json` +
  `standings.json`. Also computes a **real power rankings draft** (not just a standings copy -
  see methodology below) into `power_rankings.json`, and creates a blank `commentary.json` stub.
  Both are written `published: false` for Hunter to review/edit before they go live - the script
  never writes opinion or prose, only data-driven blurbs.

  **Power ranking methodology** (Hunter's own formula from a previous season, matched exactly):
  a weighted composite of four factors, tunable via `POWER_WEIGHTS` at the top of the script
  (default weights: expected wins 40%, record 30%, points scored 20%, recent form 10%):
  - `win_pct` - season record (ties = 0.5 win)
  - `avg_points` - season PPG, min-max normalized against the field so it's on the same 0-1
    scale as the other three factors
  - `expected_win_pct` - "all-play" record: each week, a team's score is compared against
    *every* other team's score that week (not just their real opponent) - beating a lower score
    = 1 expected win, tying = 0.5. This is the standard luck-adjustment: a team with a losing
    record but high expected wins has just run into the league's best scores repeatedly; a team
    with a winning record but low expected wins has been winning on weak scores. Verified with a
    constructed 4-team/3-week test case before shipping: a team with a losing 1-2 record but the
    league's best scores correctly ranked #1, while a team with the best 2-1 record but the
    league's worst scores correctly ranked last.
  - `recent_form` - last 3 games (fewer early in the season), blending recent win% with recent
    normalized PPG, so a current hot or cold streak shifts a team's rank independent of their
    full-season record.
  Trend arrows compare each week's computed rank against the previous week's computed rank
  (whether or not that previous week was ever published).
- `scripts/espn_credentials.example.json` - template for the two cookies needed
  (`espn_s2`, `swid`). Real file is `scripts/espn_credentials.json`, gitignored and excluded
  from `nwl_site.zip` - **never commit or zip this file**, it's Hunter's personal login session.
  Docstring at the top of the script explains exactly how to get the cookie values from a
  browser's DevTools.
- `scripts/espn_team_map.json` (created by running `--map-teams`) - maps ESPN's numeric team IDs
  to the manager names used everywhere else on the site. One-time setup per season; needs
  credentials to populate since the team list itself requires auth.
- `.gitignore` added to the site root (didn't exist before) specifically to keep the credentials
  file out of any future git history.

**Pipeline validated end-to-end**: Hunter ran the script against real 2025 season data
(`--season 2025 --week 1`). Every score matched this site's own historically-extracted
`weekly_scores.json` exactly across all 12 managers. Auth, team mapping, and JSON output are all
confirmed working. Two bugs found and fixed along the way (see REVIEW_LOG Round 11): an SSL
certificate error common to python.org's macOS installer (fixed by bundling `certifi` and an
explicit `ssl.SSLContext`), and a path-resolution bug where running the script from inside its
own folder wrote output to the wrong directory (fixed with `Path(__file__).resolve().parent`).

**`pages/season-2026.html` built** - four tabs (Matchups, Standings, Power Rankings, Commentary),
each with a week selector where relevant. Power Rankings and Commentary only ever show entries
where `published: true` - the script's auto-drafted stubs stay invisible to site visitors until
Hunter reviews and flips that flag. All four tabs have a real empty state ("Check back once the
2026 season kicks off in September") rather than looking broken when `data/season_2026/` doesn't
exist yet, via a new `loadDataSafe()` helper in `shared.js` that returns a fallback instead of
throwing on a 404.

**Visually separated from the historical archive, per Hunter's request:**
- New `--live` amber accent color (`style.css`), used *only* in this section - every other page
  keeps the existing blue/red brand palette untouched.
- Nav: `Home · 2026 Season | Hall of Fame · Managers · ...` - Home and 2026 Season (amber, small
  glowing dot) come first, then a vertical divider, then the historical links (`shared.js`).
- The page itself uses a `.season-banner` treatment (amber border/glow) instead of the plain
  `<h1>` + `<p>` every archive page uses, and its internal tabs highlight amber instead of blue.
- Homepage got a matching amber `.season-banner` linking to the page, placed *above* and visually
  distinct from the "Explore the archive" card grid - not just a 7th card blended into the
  historical six.
